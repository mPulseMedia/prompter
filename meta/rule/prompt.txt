NOTE: This file defines the process for executing prompts from the meta files.

1. Commands
    a. run
        1. Overview: Execute next prompt after prompt_marker
        2. Steps:
            a. Find prompt_marker position in app_prompt files
            b. Execute next prompt after prompt_marker per Execution Rules
            c. Update prompt_marker location per prompt_marker rules

    b. reset
        1. Overview: Reset project to initial state
        2. Steps:
            a. app directory
                1. Delete all files and directories in app/ except directory itself
            b. prompt_marker
                1. Remove from current location
                2. Place in initial position per prompt_marker rules
            c. index files
                1. Reset ALL index files (*_index.txt) to empty state
                2. Each index file must:
                    a. Keep header structure
                    b. Include only "01. " line
                    c. End with divider line
                3. Affected files:
                    a. app_prompt/app_index.txt
                    b. meta/meta_index.txt

2. prompt_marker
    a. Format
        1. Text: "/////////////////////////////// YOU ARE HERE ///////////////////////////////"
        2. One blank line before and after marker
    b. Run Behavior
        1. After each prompt execution:
            a. Move marker after executed prompt
            b. Move to next file if current file complete
        3. Always marks next prompt to execute
    c. Reset Behavior
        1. CRITICAL: Must be placed in initial position
        2. Must be in first prompt file in app_prompt/
        3. Must be after NOTE section
        4. Must be before first prompt
        5. Must have blank lines before and after
    d. File Rules
        1. Must be in app_prompt/ directory

3. Execution Rules
    a. Overview: Execute prompt according to files in rule folder
        1. Steps:
            a. Review existing codenames in code files and app_prompt/app_index.txt
            b. Select new codenames per codename.txt rules
            c. Apply format_code.txt rules to changed files
            d. Update app_index.txt with new/changed/deleted codenames

    b. Terminal Rules
        1. File Operations
            a. Use delete_file for ALL deletions
            b. Use edit_file for ALL creations/modifications
            c. NEVER use terminal for file operations
        2. Command Rules
            a. NEVER use commands requiring [yn] input
            b. ALWAYS pipe to cat if pager possible
            c. NEVER use commands that show pager
            d. ALWAYS use -f or --force flags for rm, mv, cp
            e. NEVER use interactive flags (-i) on any command
            f. ALWAYS use non-interactive flags (-n) when available
            g. ALWAYS use --no-interactive when available
            h. ALWAYS use -y for commands that might prompt
            i. NEVER use commands that might wait for input
            j. ALWAYS verify command flags prevent prompts before running

4. Chat Output Rules
    a. Format
        1. Be brief and direct
        2. Use codenames to describe changes
        3. Group related changes together
        4. Skip obvious details
    b. Change List
        1. Show all new codenames
        2. Show all modified codenames
        3. Show all deleted codenames
        4. Group by type (_file, _dir, _func, _var)
