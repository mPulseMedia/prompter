NOTE: This file defines core naming conventions for all project identifiers.

1. Terminology
    1. codename - unique name for a concept, object, file, folder, function, variable in the project
    2. term - one or more terms make up the codename, separated by _
    3. root - the first term in a codename
    4. root_list - the list of all roots used throughout the project

2. Format Rules
    1. Use snake_case
    2. Keep under 4 terms
    3. No special characters
    4. Use common abbreviations consistently

3. Scope Rules
    1. Apply to all identifiers
    2. Include files, directories, functions, variables, and anything cursor.ai can name
    3. No exceptions allowed

4. Path Rules
    1. Create codename from file path, ignoring extensions and numbers
    2. Use established roots to shorten path names
