NOTE: This file defines rules for managing and formatting rules in the project.

1. manage
    a. add
        1. Check existing rules before creating new ones
        2. Add to existing rule file if topic matches
        3. Create new rule file only if necessary
    b. update
        1. Review rules for clarity and precision
        2. Remove vague or marketing language
        3. Make rules specific and actionable

2. structure
    a. file
        1. Start with NOTE section
        2. Group related rules together
        3. Use four spaces for indentation
        4. End with divider line
    b. section
        1. Use clear category names
        2. Keep hierarchy logical
        3. Maintain consistent depth
