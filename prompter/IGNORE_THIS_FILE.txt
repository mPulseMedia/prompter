NOTE: This file defines rules specific to app development and maintenance.

• update
  • app_index
    • Must update after any code change
    • Must list all app files
    • Must show file relationships
    • Must use space formatting
  • format
    • One file per line
    • Sort alphabetically
    • Use spaces for matching prefixes
    • Show full path from app root
  • example
    • First line: app/css/style.css
    • Second line: app/css/theme.css
    • Shows as:
      • app/css/style.css
      •        theme.css
  • check
    • Verify all files listed
    • Verify alphabetical order
    • Verify space formatting
    • Verify no duplicates

• process
  • steps
    • Make code changes
    • Update app_index.txt
    • Verify index format
    • Commit both changes
  • timing
    • Update index immediately
    • Don't delay index update
    • Don't batch index updates
  • scope
    • Include all app files
    • Include all app folders
    • Include all app types
