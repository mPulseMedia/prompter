NOTE: This file defines all rules for picking codenames in the project.

• format
  • case
    • Use snake_case for all names
    • Write all letters in lowercase
    • Use underscores between terms
    • No spaces or special characters
  • length
    • Keep each term under 15 letters
    • Keep names short and clear
    • Use single words for terms
    • Avoid unnecessary length

• terms
  • action
    • Use present tense verbs
    • Keep verbs simple and clear
    • Use approved action roots
    • Match verb to function
  • object
    • Use singular nouns
    • Keep nouns descriptive
    • Use approved object roots
    • Match noun to purpose

• roots
  • types
    • Use only approved root types
    • Use verb roots for actions
    • Use noun roots for objects
    • Follow root patterns
  • usage
    • Match root to its purpose
    • Keep roots consistent
    • Use proper root level
    • Avoid custom roots

• context
  • scope
    • Match codename to visibility level
    • Match codename to its function
    • Match codename to its position
    • Keep scope clear in the name
  • location
    • Match codename to project location
    • Match codename to file structure
    • Match codename to hierarchy
    • Keep location clear in name

• path
  • generation
    • Create codename from folder path
    • Append each folder name with underscore
    • Ignore numbers and extensions
    • Keep path structure clear
  • example
    • Path: prompter/rule/command.txt
    • Codename: prompter_rule_command
    • Path: app/css/style.css
    • Codename: app_css_style

• patterns
  • action
    • get_user_data - Retrieve user info
    • set_config_value - Update settings
    • find_active_users - Search users
    • update_user_status - Change state
  • object
    • user_name - User identifier
    • config_value - Setting value
    • active_users - Current users
    • current_status - State info

• validation
  • check
    • Verify name follows format rules
    • Confirm root term is approved
    • Ensure terms are appropriate
    • Validate against patterns
  • scope
    • Check name matches purpose
    • Verify location is correct
    • Confirm hierarchy is clear
    • Validate context is proper
