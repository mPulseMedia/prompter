NOTE: This file defines rules for managing and formatting rules in the project.

• manage
  • add
    • Check existing rules before creating new ones
    • Add to existing rule file if topic matches
    • Create new rule file only if necessary
    • Keep related rules together in same file
  • update
    • Review rules for clarity and precision
    • Remove vague or marketing language
    • Make rules specific and actionable
    • Keep rules focused on exact requirements

• precise
  • language
    • Use clear, non-technical terms
    • Avoid marketing or promotional language
    • Be specific about requirements
    • State exact conditions and limits
  • scope
    • Define clear boundaries for each rule
    • Specify when and where rules apply
    • List all exceptions explicitly
    • State exact consequences of violations
  • format
    • Use consistent structure for all rules
    • Keep bullet points focused and clear
    • Use active voice for requirements
    • Include examples for complex rules

• bullet
  • length
    • Keep top-level bullets short and focused
    • Use second-level for subcategories
    • Allow 50-70 chars for detailed points
    • Break long points into multiple points
  • content
    • Start with action or requirement
    • Include specific conditions
    • State exact limits or boundaries
    • Add examples when needed

• structure
  • file
    • Start with NOTE section
    • Group related rules together
    • Use consistent indentation
    • End with divider line
  • section
    • Use clear category names
    • Keep hierarchy logical
    • Maintain consistent depth
    • Group by function or purpose
